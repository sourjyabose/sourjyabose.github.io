## 11) Multi-threaded distributed scraping architecture explain karo.

**Answer:**

“Architecture mein multiple layers the:

### 1. Task Producer

Ye scraping jobs create karta tha.

Example:
scrape amazon products

### 2. Message Broker

Redis use kiya as queue.

### 3. Workers

Multiple Celery workers parallel tasks process karte the.

### 4. Scraper Engine

Selenium / Playwright / Scrapy depending on target site.

### 5. Data Storage

MongoDB / MySQL / CSV.

Multi-threading use hota tha ek worker ke andar concurrency ke liye.

Distributed means multiple machines pe workers run ho sakte hain.”

## 12) Celery aur Redis ka use exactly kaise kiya?

**Answer:**

“Redis ko message broker aur kabhi cache ke tarah use kiya.

Celery asynchronous task queue hai.

Flow:

Application → Task enqueue → Redis queue → Celery worker picks task → executes scraper → stores result.

Benefits:

* parallel execution
* retries
* task scheduling
* scalability”

---

## 13) Scrapy aur Selenium mein difference kya hai? Kab kaunsa use karoge?

**Answer:**

Scrapy:

* Fast
* Async framework
* HTML parsing efficient
* Static websites ke liye best

Selenium:

* Real browser automation
* JavaScript render kar sakta hai
* Slow comparatively

Use case:

Static site → Scrapy

Dynamic JS-heavy site → Selenium

Agar API mil jaye → requests library”

---

## 14) Playwright aur Selenium mein kya difference hai?

**Answer:**

Playwright:

* Faster
* Modern
* Better auto-waiting
* Multiple browser contexts
* Better for modern apps

Selenium:

* older
* stable
* widely used

Agar modern SPA scrape karna hai toh Playwright prefer karungi.”

## 18) Robots.txt kya hota hai?

**Answer:**

“Website ka instruction file hota hai jo bots ko batata hai:

* kya access kar sakte hain
* kya disallow hai

```txt

```

Ye legal rule nahi hota always, but ethical guideline hota hai.”

## 19) XPath aur CSS Selectors mein difference?

**Answer:**

CSS selectors:

Fast aur simpler.

Example:

```css
div.product span.price
```

XPath:

More powerful.

Example:

```xpath
//div[@class='product']//span[text()='₹100']
```

XPath can traverse upward.

CSS cannot.”
